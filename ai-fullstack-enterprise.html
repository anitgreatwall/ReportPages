L1 · INFRASTRUCTURE

基础层：模型与存储

一切 AI 能力的地基。大语言模型提供智能核心，向量数据库实现语义检索，推理引擎控制速度与成本——这层的选择决定了整个系统的天花板。

🧠
LLM
大语言模型
整个系统的智能核心。GPT-4、Claude、Gemini 等模型通过海量文本训练，具备语言理解、推理、生成能力。企业选型需权衡：能力、成本、数据安全、延迟。
OpenAIClaudeGeminiLlama
🔢
EMBEDDING
向量嵌入
将文本、图片、音频转换为高维向量数字表示，使计算机能计算语义相似度。"苹果手机"和"iPhone"会被嵌入为接近的向量，是 RAG 和语义搜索的基础。
text-embedding-3sentence-transformers
🗃️
VECTOR DATABASE
向量数据库
专为存储和检索向量而设计的数据库。能在毫秒内从千万文档中找到语义最相近的内容，是企业知识库的核心存储引擎。
PineconeWeaviateChromaQdrant
🎯
FINE-TUNING
微调训练
在通用模型基础上，用企业私有数据继续训练，使模型学会行业术语、企业风格、专业知识。让"通才"变成你的行业"专才"。
LoRAQLoRARLHF
⚡
INFERENCE ENGINE
推理引擎
控制模型运行效率的底层引擎。通过量化、批处理、缓存等技术将推理速度提升数倍，同时大幅降低成本。生产环境中不可忽视的关键组件。
vLLMTGITensorRT
💾
CONTEXT WINDOW
上下文窗口
模型单次能处理的最大文本量（token 数）。决定了能喂给模型多少上下文信息，直接影响长文档处理、多轮对话、复杂任务执行的能力边界。
128K tokens200K tokens
L2 · PERCEPTION

感知层：读懂世界

让 AI 从"只能看文字"升级为能理解文档、图片、语音等多元信息。RAG 解决知识时效问题，多模态打破单一输入限制，Document AI 让非结构化数据可用。

🔍
RAG
检索增强生成
Retrieval-Augmented Generation。回答问题时先检索相关文档，再将文档内容注入提示词让模型作答。解决模型"知识过时"和"幻觉"两大核心问题，是企业知识库的标配方案。
语义检索知识增强私域数据
🌈
MULTIMODAL
多模态
同时处理文本、图片、音频、视频等多种类型输入的能力。上传一张产品图片就能分析质量问题，发送语音就能生成文字摘要——让交互更自然。
VisionAudioVideo
👁️
OCR / VISION
视觉理解
从图片和扫描件中提取文字，理解图表、表格、手写内容。结合 AI 不只是识别字符，还能理解图片语义——看到产品缺陷图就能输出质检报告。
Document OCRTable Parsing
📋
DOCUMENT AI
文档智能
专门处理 PDF、Word、Excel、合同、发票等非结构化文档的 AI 能力。自动提取关键字段、识别文档类型、理解布局结构，将文档转化为可操作的结构化数据。
合同解析发票识别表单提取
🎤
ASR / TTS
语音识别与合成
ASR（自动语音识别）将语音转文字，TTS（文字转语音）将文本朗读。赋予 AI 系统"耳朵"和"嘴巴"，支撑语音客服、会议纪要、语音指令等场景。
WhisperAzure Speech
🗂️
CHUNKING STRATEGY
文档分块策略
将长文档切割成适合检索的小块的策略。切太大信息冗余，切太小缺乏上下文。语义分块、递归分块、滑动窗口等策略直接影响 RAG 检索质量。
语义分块滑动窗口
L3 · REASONING

推理层：思考与决策

AI 不只是"查字典"，更要能思考、规划、决策。这一层定义了 AI 如何处理复杂问题——从单步问答，到多步规划，到自主完成开放性任务。

🤖
AGENT
智能体
能自主感知目标、制定计划、调用工具、执行多步骤任务的 AI 系统。区别于简单问答——Agent 会主动"做事"，遇到问题会自动调整策略，直到目标完成。
自主决策工具调用循环执行
💭
CHAIN OF THOUGHT
链式推理 (CoT)
让模型在给出答案前"把思考过程写出来"的技术。显著提升复杂问题的准确率——就像人类解数学题时打草稿，逐步推导比直接猜答案准确得多。
逐步推导思维链
⚛️
ReAct
推理 + 行动
Reasoning + Acting。让 AI 交替进行"思考"和"行动"：想→查→看结果→再想→再查。比纯推理更准确，比纯执行更灵活，是 Agent 设计的核心范式。
ThoughtActionObservation
🧩
MEMORY SYSTEM
记忆系统
AI 的记忆分四种：短期记忆（对话历史）、长期记忆（向量存储）、情节记忆（过去经历）、程序记忆（固化技能）。记忆架构决定了 AI 能否"越用越聪明"。
短期记忆长期记忆情节记忆
🗺️
TASK PLANNING
任务规划
将复杂目标分解成有序子任务序列的能力。包括任务依赖分析、优先级排序、资源分配。好的规划能让 Agent 高效并行执行而不乱套。
任务分解依赖图并行执行
🌳
TREE OF THOUGHTS
思维树 (ToT)
让 AI 探索多条推理路径，像棋手分析多步棋一样。对于有多种解法的复杂问题，ToT 能找到最优路径，避免单线推理陷入死胡同。
多路径探索剪枝搜索
L4 · CAPABILITIES

能力层：工具与技能

赋予 AI 具体行动能力的工具箱。Function Calling 让 AI 调用 API，Code Interpreter 让 AI 跑代码，Skills 封装最佳实践——从"能想"到"能做"的关键桥梁。

⚡
SKILLS
技能模块
封装特定任务最佳实践的可复用能力包。当 AI 遇到"生成 PPT"任务时，自动加载 PPTX Skill，按照预定义的高质量模板和流程执行，而非每次从零摸索。
可复用可组合领域专精
🔧
FUNCTION CALLING
函数调用
AI 识别用户意图后，自动选择并调用预定义函数（API）的能力。模型决定"调用什么函数、传什么参数"，系统负责真正执行——AI 与外部世界交互的标准接口。
结构化输出API 桥接
💻
CODE INTERPRETER
代码解释器
让 AI 在沙箱环境中编写并运行代码，处理数据分析、图表生成、文件转换等任务。AI 能根据运行结果自动调试，直到得出正确输出。
Python数据分析沙箱执行
✍️
PROMPT ENGINEERING
提示词工程
系统性设计输入提示词以最大化模型输出质量的学科。包括角色设定、少样本示例、思维链引导、输出格式约束等技术——同样的模型，提示词好坏可使结果天壤之别。
System PromptFew-shotCoT
🎭
STRUCTURED OUTPUT
结构化输出
强制模型按照预定义的 JSON Schema 或格式输出，而非自由文本。确保下游系统能可靠地解析 AI 输出，是生产级 AI 集成的必要条件。
JSON Schema强类型
🔄
TOOL USE
工具调用
赋予 AI 访问外部工具的能力：搜索引擎、计算器、日历、数据库查询等。每个"工具"都是 AI 能力的延伸，工具集的丰富程度决定了 AI 的行动半径。
Web SearchCalculatorDB Query
L5 · ORCHESTRATION

编排层：协调全局

当单个 Agent 不够用时，需要编排层来协调多个 Agent 和工具协同工作。Workflow 定义蓝图，Orchestrator 统筹调度，Multi-Agent 分工协作——企业级复杂任务的指挥中枢。

🔄
WORKFLOW
工作流
将多个步骤、判断、工具调用串联成有序流程的蓝图。定义触发条件、执行顺序、分支逻辑和错误处理。从邮件自动分类到报告自动生成，都由 Workflow 来编排。
DAG条件分支并行执行
🏛️
MULTI-AGENT
多智能体协作
多个专业化 Agent 分工协作完成复杂任务。如"研究 Agent"负责搜集信息，"分析 Agent"负责数据处理，"写作 Agent"负责生成报告——各司其职，合力完成单 Agent 无法胜任的任务。
分工协作专业化并行
🎛️
ORCHESTRATOR
任务协调器
多 Agent 系统的"项目经理"。负责分配任务、监控进度、处理失败、整合结果。好的 Orchestrator 让复杂任务执行有条不紊，如同一支训练有素的团队。
LangGraphCrewAIAutoGen
🔁
PIPELINE
处理管道
数据或任务按序流经一系列处理节点的架构模式。每个节点做一件事并将结果传给下一节点。数据预处理→AI 推理→后处理→输出，构成完整的 AI 处理流水线。
流式处理批处理
📡
EVENT BUS
事件总线
系统间异步通信的中枢。当某事件发生（如新订单、用户投诉），事件总线将消息广播给所有订阅方，触发相应的 AI 工作流。实现系统解耦与实时响应。
KafkaRedis Pub/Sub
🔃
FEEDBACK LOOP
反馈循环
AI 执行结果反馈回系统，用于评估质量、调整策略、优化模型的闭环机制。没有反馈循环的系统只能"用一次"，有反馈的系统才能"越用越好"。
RLHF持续优化
L6 · CONNECTIVITY

连接层：打通数据孤岛

AI 系统的价值在于能访问并操作真实数据。MCP 提供标准化接口，API 对接现有系统，ETL 整合历史数据，Webhook 实现实时触发——让 AI 真正融入企业数字生态。

🔌
MCP
模型上下文协议
Anthropic 推出的开放标准协议（2024）。统一了 AI 模型与外部工具/数据源的交互方式，就像 USB 接口一样——有了 MCP，一套代码可接入数百种工具，无需为每个工具单独开发集成。
开放协议双向通信生态标准
🌐
API / REST / GraphQL
接口对接
通过标准化接口与企业现有系统集成：ERP、CRM、数据库、SaaS 工具等。AI 读取数据、写入结果、触发业务流程——无需改造现有系统，即可注入 AI 能力。
RESTGraphQLgRPC
🪝
WEBHOOK
事件钩子
当特定事件发生时，系统主动推送通知到指定地址。与轮询相反——不是 AI 不停"问有没有新数据"，而是数据变化时系统主动"告诉"AI，实现实时、高效的事件驱动。
事件驱动实时触发
🔀
ETL / ELT
数据集成管道
Extract-Transform-Load。将分散在各业务系统的数据抽取、清洗、转换后导入统一数据湖，为 AI 提供可靠的数据燃料。数据质量是 AI 输出质量的先决条件。
AirbytedbtFivetran
📨
MESSAGE QUEUE
消息队列
异步消息传递的缓冲层。AI 任务提交到队列，由工作节点异步处理，避免高峰期系统崩溃。同时提供任务持久化、重试机制，确保不丢失任何请求。
KafkaRabbitMQSQS
🔐
AUTH / IAM
身份与权限管理
控制 AI 系统能访问什么数据、能执行什么操作的权限框架。AI Agent 需要精细的权限边界——能读销售数据但不能删除，能发邮件但不能转账。最小权限原则是安全基石。
OAuthRBAC最小权限
L7 · EXECUTION

执行层：真实世界操控

AI 最终需要在真实环境中行动——点击按钮、填写表单、操作软件、运行脚本。这一层是 AI 与物理世界的接触面，决定了自动化能触达多深的业务操作。

🦾
RPA / ROBOT
流程自动化机器人
模拟人类操作计算机界面的技术：移动鼠标、点击按钮、输入文字、截图。无需 API 即可操作任何软件，是处理遗留系统（Legacy）的利器。结合 AI 后能理解上下文、灵活应变。
UiPathAutomation Anywhere
🖥️
COMPUTER USE
电脑操控
新一代 AI 原生的电脑操控能力（如 Claude's Computer Use）。AI 通过截图理解屏幕内容，自主决定下一步操作——不依赖预录制脚本，能处理动态变化的界面。
Claude CU视觉定位自适应
🌐
BROWSER AUTOMATION
浏览器自动化
控制浏览器自动执行网页操作：登录、填表、抓取数据、点击、滚动。比 RPA 更专注于 Web，可精确控制 DOM 元素，是 Web 数据采集和网页操作的首选。
PlaywrightPuppeteerSelenium
⌨️
SHELL / CLI EXEC
命令行执行
AI 在服务器上执行 Shell 命令：文件操作、进程管理、系统配置、脚本运行。让 AI 成为真正的 DevOps 助手，能自动化运维任务、处理服务器突发情况。
BashPythonDevOps
📱
MOBILE AUTOMATION
移动端自动化
在手机 App 上执行自动化操作：点击、滑动、输入。适用于没有开放 API 的移动应用，可实现移动端业务流程的 AI 自动化。
AppiumiOS/Android
📦
SANDBOX / CONTAINER
沙箱与容器
为 AI 提供安全隔离的执行环境。AI 运行代码、操作文件、访问网络都限制在沙箱内，防止意外影响生产系统。Docker 容器化让每次执行都在干净、可复现的环境中进行。
DockerE2B安全隔离
L8 · GOVERNANCE

治理层：安全与可控

生产级 AI 系统必须可信赖、可观测、可审计。Guardrails 防止 AI 越界，Observability 让系统透明可查，Evals 持续验证质量——没有治理层的 AI 是不敢上线的 AI。

🛡️
GUARDRAILS
安全护栏
约束 AI 行为边界的安全机制：过滤有害输出、阻止越权操作、检测 Prompt 注入攻击。企业部署 AI 必须设置护栏——就像电梯有超重保护，AI 也需要安全兜底。
内容过滤Prompt 防注入输出审核
📊
OBSERVABILITY
可观测性
对 AI 系统运行状态的全面监控：每次调用的延迟、Token 消耗、错误率、用户满意度。没有可观测性，AI 系统如同黑盒，出了问题无从排查、无法优化。
LangSmithHeliconeTracing
🧪
EVALS
评估体系
系统性衡量 AI 输出质量的方法论。包括：自动化评估（准确率、F1 分数）、人工评估、LLM-as-Judge（用 AI 评估 AI）。没有评估，就不知道系统在变好还是变坏。
自动评估LLM-JudgeA/B测试
📝
AUDIT LOG
审计日志
完整记录 AI 系统每一次操作的时间、内容、结果和执行者。满足合规要求，支持事后溯源——出现纠纷时能精确还原"AI 当时做了什么、为什么这么做"。
合规可溯源不可篡改
💰
COST MANAGEMENT
成本管控
AI Token 消耗即成本，大规模使用时费用可能急剧上升。成本管控包括：Token 预算限制、模型路由（简单任务用便宜模型）、缓存复用、批处理优化。
Token 预算模型路由缓存
🔒
DATA PRIVACY
数据隐私保护
确保敏感数据不泄露给外部模型的机制：PII 脱敏、私有化部署、数据分级管理。企业数据是核心资产，AI 系统必须在隐私合规框架内运行。
GDPRPII脱敏私有化部署
L9 · DELIVERY

交付层：触达用户

AI 能力最终必须以用户友好的形式呈现。Copilot 嵌入工作流，Chatbot 提供对话入口，AI Gateway 统一管理 API，知识库沉淀企业智慧——这层决定了 AI 如何被真正用起来。

🧑‍✈️
AI COPILOT
智能副驾
嵌入到现有工作界面的 AI 助手，在用户工作时实时辅助：写文档时提供建议、写代码时补全、分析数据时给出洞察。不替代用户决策，而是让每个员工都有专属 AI 助理。

SCENARIO 01
供应链异常自动处理
触发：供应商发来一封"延迟交货"邮件 → 全链路自动处置
L2 · PERCEPTION
Document AI
解析供应商邮件，提取关键信息：供应商名称、延误天数（7天）、受影响 SKU（32个）、原因（港口罢工）
EXTRACT
L3 · REASONING
Agent + CoT
链式推理评估影响：查询库存 → 预测缺货日期 → 计算损失金额 → 生成 3 套应对方案（备用供应商、空运、客户沟通）
ANALYZE
L6 · CONNECT
MCP + API
通过 MCP 同时查询：ERP 库存系统（剩余量）、备供商数据库（报价和产能）、物流 API（空运时效和价格）
QUERY
L5 · ORCHESTRATE
Workflow + Multi-Agent
Orchestrator 并行分配：采购 Agent 联系备用供应商询价，客服 Agent 起草客户通知，财务 Agent 生成损失预估报告
COORDINATE
L7 · EXECUTE
RPA + Browser Auto
Robot 自动操作采购系统：创建紧急采购单、登录备供商 Portal 提交订单、在 ERP 更新预计到货日期，全程无需人工介入
AUTOMATE
L8 · GOVERN
Audit + Guardrails
全程记录审计日志，Guardrails 检查：采购金额是否超权限阈值（超过则暂停等待人工审批），合规报告自动归档
GOVERN
L9 · DELIVER
Dashboard + Copilot
管理驾驶舱实时显示处置进度，Copilot 向采购总监推送一页决策简报，人工只需确认最终方案，整体响应时间从 4 小时压缩至 12 分钟
✓ DONE
SCENARIO 02
合同智能审查系统
触发：律所上传一份 200 页采购合同 → AI 完成合规审查
L2 · PERCEPTION
Document AI + OCR
解析 200 页 PDF，识别合同结构（条款编号、标题、正文），提取关键字段：甲乙方、金额、期限、违约条款、知识产权归属
PARSE
L1 + L2 · INFRA
RAG + Vector DB
将合同内容向量化，与企业法律知识库（历史合同、法规库、判例库）进行语义匹配，定位相似条款和潜在风险
RETRIEVE
L3 · REASONING
Agent + Tree of Thoughts
逐条分析合同风险：发现 12 处高风险条款（责任上限过低、仲裁条款不利、自动续约陷阱），每条附上法律依据和修改建议
ANALYZE
L4 · SKILLS
Skills + Structured Output
加载法律文书生成技能，按结构化格式输出审查报告：风险等级（高/中/低）、原始条款、问题说明、修改建议、参考法条
GENERATE
L9 · DELIVER
AI Copilot + Knowledge Base
律师通过 Copilot 界面与 AI 对话深入讨论风险点，一键导出 Word 审查报告，本次合同自动归档至知识库，丰富未来的检索样本
✓ DELIVER
SCENARIO 03
销售线索智能转化
触发：官网新增一条询盘表单 → AI 全程跟进直到成单
L6 · CONNECT
Webhook + ETL
官网表单触发 Webhook，实时推送到 AI 系统。ETL 自动从多个数据源丰富线索信息：爬取公司官网、LinkedIn、查工商数据
TRIGGER
L3 · REASONING
Agent + Memory
Agent 综合判断线索质量：公司规模 500 人、融资 B 轮、痛点与产品高度匹配，评分 92 分，记入长期记忆供后续对话调用
SCORE
L4 · SKILLS
Skills + Prompt Eng
加载销售邮件技能，基于客户行业背景生成个性化开发信，引用客户最近一次融资新闻，说明产品如何解决其扩张期痛点
PERSONALIZE
L7 · EXECUTE
Browser Auto + RPA
Robot 自动操作 CRM：创建线索卡片、录入画像数据、发送开发邮件、设置 3 天后跟进提醒，销售收到的是一份已完成前期调研的热线索
EXECUTE
L8 · GOVERN
Observability + Evals
监控开信率（68%）、回复率（31%）、转化率，通过 A/B 测试持续优化邮件策略，Evals 自动评估每封邮件质量并反馈模型微调
OPTIMIZE
SCENARIO 04
安全事件自动响应
触发：SIEM 检测到异常登录行为 → AI 自动完成威胁评估与处置
L6 · CONNECT
Event Bus + API
SIEM 通过事件总线推送告警：用户 admin@corp 凌晨 3 点从俄罗斯 IP 登录，触发 MFA 失败 5 次，AI 系统实时接收并启动响应流程
ALERT
L3 · REASONING
Agent + ReAct
ReAct 循环：查威胁情报库（该 IP 已被标记为恶意）→ 查用户历史（无海外登录记录）→ 查近期操作（刚导出大量数据）→ 威胁等级判定为 CRITICAL
ASSESS
L5 · ORCHESTRATE
Workflow + Multi-Agent
Orchestrator 并行触发：封禁 Agent（立即锁账号）、取证 Agent（收集操作日志）、通知 Agent（叫醒安全团队）同步执行
RESPOND
L7 · EXECUTE
Shell Exec + API
自动执行：禁用 AD 账号、吊销所有 Session Token、封禁源 IP、隔离可能受感染的工作站，整个封禁过程在 23 秒内完成
CONTAIN
L8 · GOVERN
Audit + Guardrails
所有操作写入不可篡改审计日志，Guardrails 确认封禁操作在授权范围内，自动生成合规事件报告，满足 ISO 27001 记录要求
AUDIT
L9 · DELIVER
Dashboard + Report
安全驾驶舱实时展示事件处置时间线，AI 自动生成事件调查报告，安全总监收到完整的攻击链分析和改进建议。MTTD 从 4 小时降至 8 秒
✓ RESOLVED
